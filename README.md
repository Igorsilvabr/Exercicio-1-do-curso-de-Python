# Exercicio-1-do-curso-de-Python

Eu comecei aprendendo a fazer o programa conversar comigo e com o jogador. Para isso, usei a função print() para colocar as perguntas e mensagens na tela e a função input() para o programa pausar e receber a resposta que eu digitava. Essas duas funções simples são o essencial para qualquer interação em um jogo.

O coração do meu mini-game foi a tomada de decisões, que fiz usando as estruturas if, elif e else. Eu usei o if para testar a condição principal (por exemplo, se a resposta estava correta ou se o jogador queria continuar). O elif me ajudou a testar uma segunda condição, como quando o jogador respondia 'n' para sair. E o else garantia que, se nenhuma das condições fosse verdadeira, o programa ainda teria uma mensagem final, como a de erro ou a de saída.

Para garantir que o meu jogo não fosse muito rígido, usei o operador in. Ele me permitiu aceitar várias respostas diferentes (como "buraco", "o buraco" ou "Buraco") como corretas para o mesmo enigma. Isso foi muito mais eficiente do que ter que escrever um monte de condições if separadas.

O conceito mais importante que eu aprendi foi o Controle de Fluxo e o Aninhamento, que é feito com a Indentação (os espaços no início da linha). Eu descobri que, para o jogo realmente parar quando eu digitava 'n', todo o código do próximo enigma precisava estar dentro do if de continuação do enigma anterior. Isso garante que o programa só prossiga se a condição de 'sim' for verdadeira, ou ele segue para a mensagem de elif e termina. Eu usei essa técnica para encadear todos os três enigmas e a brincadeira final.
